## 1. 语音：VOICEVOX 还有没有更好的替代？要不要加更多声音？

### 结论先行
- **继续以 VOICEVOX 为主。** 对你（弱点是**音高/读音**）来说，VOICEVOX 最大价值不是「最自然」，而是它的 `/audio_query` **能逐拍看到并修改重音（downstep）**——这是其它免费引擎给不了的可控性，也正是上面校验流程能成立的原因。
- **最值得加的免费升级（针对你的弱点）= 音高可视化。** 用 OJAD/Suzuki-kun 那种「高低线」，配合 fugashi+UniDic 的 `aType`（重音核位置），在每条句子旁画一条 高─低 的音调线。**边听 VOICEVOX、边看降调点**，对「读音/声调」这个弱项是杠杆最大的一招，且免费离线。
- **想要更自然 + 更多花样（低风险）= 加 AivisSpeech。** 它把 Style-Bert-VITS2 套了一层**和 VOICEVOX 完全兼容的 API**——我们的脚本几乎不用改就能多接一个更自然、更有感情的引擎。
- **「更多声音 for fun」很简单**：VOICEVOX 自带 20+ 角色、100+ 风格（普通/甜/傲娇/耳语…），每个有数字 `speaker id`。`GET /speakers` 列出全部，生成时循环几个 id 就能出多套音色。

### 候选对比（精简版）

| 引擎 | 免费 | 要 key | 本地/云 | 日语自然度 | **重音可控** | 接入难度 | 备注 |
|---|---|---|---|---|---|---|---|
| **VOICEVOX**（现用） | ✅ | 否 | 本地 | 高 | **最强（可逐拍编辑）** | 已接 | 部分角色需署名 |
| **AivisSpeech** | ✅ | 否 | 本地 | **最自然** | 规则级，不可逐拍 | **几乎零改**（API 兼容） | 模型 license 各异 |
| COEIROINK / SHAREVOX | ✅ | 否 | 本地 | 高 | 可逐拍（同 VV 系） | 同 VV | 角色较少 |
| **edge-tts** | ✅ 免 key | 否 | 云 | 很高 | ❌ 不可控、偶错 | 极易（直接出 MP3） | 非官方、ToS 灰区 |
| Piper / Kokoro | ✅ | 否 | 本地 | 日语中等 | ❌ | 易 | 日语非强项 |
| Google/Azure/OpenAI/Polly TTS | 免费额度 | ✅ | 云 | 很高 | ❌ | 易 | 适合「换个真人腔」的额外音色 |
| gTTS | ✅ | 否 | 云 | 低、单声 | ❌ | 易 | 不推荐 |

> **避免当主力**：edge-tts/云 TTS（无法控重音，对「学声调」不可靠）；要用就当「额外好玩音色」在生成阶段批量产出。

### 关于「真人专业录音」
你提到将来想让专业人士录音作为「真正可信源」——这是对的方向，TTS 永远无法 100% 保证声调。我们的架构已经为此留好了路：**只要用同名文件覆盖** `audio/d17_s7.mp3`，无需改任何代码或 manifest，站点就会播放真人版。可以从「名句 / 课文段落」这种高价值内容开始众包/外包录音。

## 2. AI 学习助手（AI Agent）

### 你的需求拆解
- 嵌在学习界面、**限定日语学习**、以当前课程为底、可延伸；
- 能回答**跨课的个性化疑问**（如「〜ています 的 て」vs「顔を洗ってから 的 て」）；
- **打字 / 语音**输入，支持**中日英混说**（像 Typeless）；
- 基础版＝文字；进阶版＝像 Siri/Jarvis 的**语音对话**。

### 架构选型（静态站、无后端、GitHub Pages）

| 方案 | 怎么跑 | 成本 | 安全 | 工作量 | 质量 |
|---|---|---|---|---|---|
| **(a) 纯浏览器 BYOK** | 你在设置里粘贴自己的 API key（存 localStorage），JS 直接 fetch 调用 | $0 基建，按量付费 | key 暴露在本机——**单人自用可接受**（设消费上限） | **最低** | 旗舰模型，最好 |
| (b) 免费 serverless 代理 | GitHub Pages 调一个 Cloudflare Worker（免费 ~10万次/天），key 藏服务端 | $0（免费档） | 最好 | 中（多一处部署） | 旗舰模型 |
| (c) 浏览器本地模型 | WebLLM/transformers.js 下载模型在本地推理 | $0 | 完美 | 中高 | **最弱**（中日英细腻语法问答不够） |

**推荐：Phase 1 用 (a) 纯浏览器 BYOK。**
- **CORS 现实**：Anthropic Claude **官方支持**浏览器直连（`anthropic-dangerous-direct-browser-access: true`）；OpenAI 也可（`dangerouslyAllowBrowser`）。**这两家最稳**，且对你这种中日英精准解释，Claude/GPT 质量最好。
- **Gemini 免费额度大**但浏览器 CORS 最不稳（兼容端点被 CORS 拦）——如果你就想白嫖 Gemini 免费档，那就上 (b) 一个 Cloudflare Worker（~30 行）绕过 CORS 并藏 key。

### 怎么「懂当前课程 + 跨课引用」（grounding）
- 课程只有 30 天、数据已是结构化 JS——**直接把当前课 + 一份「全 30 课索引」（每课的语法点/关键词）塞进 system prompt 就够了**，不需要向量库。
- **「这个语法点最早出现在第几课」用代码算，不靠 AI 猜**：扫一遍 `LESSONS` 建一个 `语法点 → 最早出现 day` 的映射，让回答里直接给出可点击的「Day N」链接。可信、不会幻觉。

### 语音 I/O（中日英混说）
- **输入**：`webkitSpeechRecognition` 只支持单语言，**搞不定混说**——这是它出局的原因（可留作纯日语听写的兜底）。混说最佳路径＝把音频丢给**原生会 ASR 的多模态大模型**（Gemini 音频 / OpenAI gpt-4o-transcribe），它们对 code-switching 远好于浏览器原生。
- **输出**：先用免费 `speechSynthesis` 的 `ja-JP` 朗读答案即可；要更自然再上 Gemini/OpenAI TTS。
- **Jarvis 实时语音对话**：最便宜可行＝**Gemini Live API**（~$0.002–0.011/分钟，有免费档），但要配 (b) 的代理转发 WebSocket；OpenAI Realtime 能更直连浏览器但贵很多（~$0.08–0.3/分钟）。

### 借鉴 Typeless 的点
它本质是「云 ASR + LLM 清洗」的系统级听写工具。值得借鉴的交互：①**按住说话→实时转写**；②**LLM 清洗口语**（去「嗯啊」、采纳自我更正、把啰嗦口语整理成一句干净问题）——这条最值钱；③**不强制选语言**，让模型自己处理中日英混说；④**对选中文本发问**（在课文里选中一句 → 语音问「这里的 て 是什么用法」）。

### 分阶段
- **Phase 1（~1–2 天，$0 基建）**：设置面板填 Claude/OpenAI key → 打字提问 → 流式文字回答；注入当前课 + 全课索引 + 「最早出现 Day N」可点链接；system prompt 把范围锁在日语学习。可顺手加免费的 `webkitSpeechRecognition` 听写 + `ja-JP` 朗读。
- **Phase 2（~3–6 天）**：Jarvis 语音模式——Gemini Live（经 Cloudflare Worker）或 OpenAI Realtime；加 Typeless 式「按住说 + LLM 清洗」；中日英混说交给多模态模型。

## 3. 智能笔记系统（Obsidian 式）

### 你的需求拆解
- 区分「大纲要我学的」vs「我真正想/懂的」；
- 汇总「我的提问 + 老师(AI)的回答」；
- Obsidian 式**双链**（笔记 B 源自 A）；
- **课程关联**：复习某语法点时，能溯源到它**最早出现的那一课**，跳回原文。

### 核心设计（要点）
1. **两个命名空间**：课程实体（课文/语法/单词，只读，存在 `lessons.js`）与你的笔记（存 localStorage）。笔记用**稳定 ID** 链到课程实体。
2. **稳定 ID 是关键**（否则改一个字链接就断）：`g:1:ています`（语法＝day+slug）、`v:覚える`、`d:1`、`d:1:noon`。`slug()` 去掉振假名和 `〜`，再配一张极小的「别名表」处理改名；解析失败就显示**灰色断链**而非崩溃。
3. **笔记 JSON**：含 `title/body(markdown)/tags/links{notes[],course[]}/provenance{kind: manual|ai-qa, derivedFrom, ai:{question,answer,studyContext}}/createdAt…`，全部存一个 `jpn-notes` 键。
4. **双链/反链**：笔记里写 `[[标题]]` 或 `[[course:g:1:ています]]`；加载时一遍扫描建**反向索引**（不持久化），渲染「被哪些笔记引用」面板——**课程卡片上也能显示「有 N 条笔记提到这个语法点」**。
5. **编辑器**：`textarea` + `marked.js` + `DOMPurify`（CDN，无构建）；可选升级 EasyMDE。`[[` 自动补全是性价比最高的增强。
6. **存储**：纯文本笔记，localStorage（~5MB）多年够用；将来要存图片/音频再迁 IndexedDB（`idb-keyval`）。**把 `jpn-notes` 加进现有 export/import** 一行搞定；导入用**合并**（按 `updatedAt`）而非覆盖。
7. **课程溯源**：加载时建 `firstGrammar[slug] → 最早 day`；点笔记里的课程链接 → 复用现有 `STATE.day/session + showPage('daily')` 路由（和现在的 glossary 跳转同一套），滚动高亮到对应卡片。
8. **与 AI 助手联动**：一次问答点「保存为笔记」→ 自动摘要 + 自动打标签 + **自动按当时的 `studyContext` 链到对应语法点**；AI 笔记带 🤖 标记，可按「全部/我的/来自AI」过滤——正好对应你「大纲 vs 我的思考 vs AI讲解」的三分。

### 分阶段
- **Phase 1（~3.5–4.5 天）**：笔记 CRUD + `[[双链]]` + 课程链接（含「最早出现 Day N」跳转）+ 反链面板 + 标签 + provenance + 并入备份。
- **Phase 2（~2.5–3.5 天）**：`[[` 自动补全 + AI 问答存为笔记 + 力导向图谱（`force-graph`）。
