## P2 — UI / 产品体验（本轮新增维度）
### R3-2【小 bug】AI 助手面板的“当前课”标签会过期
面板开着时切换天数/页面，顶部 `#ai-ctx` 仍显示打开时那一刻（例如停在「Day 8」而正文已是 Day 1）。实际回答用的是 `STATE.day` 实时值（即 Day 1）——所以**显示与实际不一致**，会误导。
- 改：在 `send()` 时刷新 `#ai-ctx`（和注入的课文一致）；或 `showPage()` 时若面板开着就更新。

### R3-3【产品】“笔记”有 3 个入口，定位易混
现在有：顶部 📓笔记页 + 右下 🗒️速记浮窗 + 主页“速记本”卡片。三者关系对新用户不直观（什么时候用哪个？速记 vs 正式笔记的边界）。
- 建议：明确分工并在 UI 点出来——🗒️=随手草稿（自动保存）→ 整理成 Section → 📓=正式笔记库；或把入口收敛。至少在速记浮窗顶部加一句“草稿，整理后存为正式笔记”。

### R3-4【产品】小屏 chrome 过重
- 手机宽度下「每日」页眉占 **3 行**（图标导航 / 30天地图+ふりがな+翻页 / 齿轮），吃掉大量首屏；
- 两个浮动按钮（🤖 / 🗒️）在手机上**压住正文**（如“每日一句”卡片、session 提示）。
- 建议：移动端把次级控件（ふりがな/翻页/地图/齿轮）收进一个“⋯”或抽屉；两个 FAB 合并成一个可展开的悬浮菜单，减少遮挡与入口数量（当前共 5 顶栏 + 齿轮 + 2 FAB ≈ 8 个入口）。

### R3-5【产品·可选】发现性
多音色、AI、笔记都藏在设置/浮窗里。可在首次进入或主页加一处轻量引导（“试试 🔊 真人音色 / 🤖 提问 / 🗒️ 记笔记”），让新同学知道这些能力存在。

## 如何“真跑”Azure 发音评估（给作者）
我的预览是**隔离的无头浏览器**：拿不到你浏览器里存的 key，也没有麦克风/真人朗读，所以**无法在我这端复现真实打分**。两种真验证方式：
1. **最贴近真实（推荐你自己做，30 秒）**：在你自己的浏览器（localhost 打开）→ 每日·朝の朗読 → 🎤 録音して読む，读一句。
   - 成功长这样：四条分数条（正確さ/流暢さ/完整さ/抑揚）+ 总分 + 逐字绿/黄/红上色 + 可“听我的录音”A/B 对比 + 最近 3 次历史。
   - 失败会显示明确文案（如“AI 引擎出错：…检查 Key/Region”）——把文案发我即可定位。
2. **我帮你服务端深验（可选）**：你把 `Key + Region` 贴给我，我用一条已生成的课文 MP3 转 WAV、直接打 Azure 发音评估 REST，验证你的 key 有效且能返回真实分数与逐音素结构。**注意 key 会出现在对话里，跑完建议在 Azure 门户轮换该 key。**
